# Why You See `.js`, `.js.map`, and `.d.ts` Files

## TL;DR
These are **compiled TypeScript output files** that should be gitignored. You're seeing them because:

1. **`.js` files** = Compiled JavaScript from TypeScript
2. **`.js.map` files** = Source maps (for debugging)
3. **`.d.ts` files** = TypeScript declaration files
4. **`.tsbuildinfo` files** = TypeScript build cache

## What Should Be in `.gitignore`

Add these lines to your root `.gitignore`:

```bash
# Build outputs
dist/
build/
*.js
*.js.map
*.d.ts
*.tsbuildinfo

# Specific exceptions (if needed)
!*.config.js
!jest.config.js
!tailwind.config.js
!prettier.config.js

# Node modules
node_modules/
.pnpm/
```

## File Breakdown

### `.js` Files (Compiled JavaScript)
- **What**: JavaScript output from TypeScript compiler
- **Why generated**: TypeScript needs to be compiled to JavaScript for Node.js
- **Should be gitignored**: YES (regenerate on `npm run build`)
- **Example**: `users.schema.js` from `users.schema.ts`

### `.js.map` Files (Source Maps)
- **What**: Mapping files that link compiled JS back to original TS
- **Why generated**: Helps debugging - stack traces show original TS line numbers
- **Should be gitignored**: YES (regenerate on `npm run build`)
- **Example**: `users.schema.js.map`

### `.d.ts` Files (Declaration Files)
- **What**: TypeScript type definitions for compiled code
- **Why generated**: Allows other packages to use TypeScript types
- **Should be gitignored**: Depends on your build setup
  - **YES if**: You rebuild on each deploy
  - **NO if**: You're publishing to npm and need them in dist/
- **Example**: `users.schema.d.ts`

### `.tsbuildinfo` Files
- **What**: TypeScript incremental build cache
- **Why generated**: Speeds up subsequent builds
- **Should be gitignored**: YES (recreated on each build)
- **Example**: `tsconfig.tsbuildinfo`

## Build Process (Turbo)

Your monorepo uses Turbo which:
1. Runs TypeScript compiler (`tsc`)
2. Generates `.js`, `.js.map`, `.d.ts` in `dist/` or package directories
3. These should be **regenerated on each build**, not committed

```bash
# This generates all those files
pnpm run build

# They'll be automatically cleaned before building
pnpm run clean
```

---

**Bottom line**: Never commit `*.js`, `*.js.map`, or `*.tsbuildinfo` files except for configuration files like `tailwind.config.js` or `jest.config.js`.
